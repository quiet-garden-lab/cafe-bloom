# Codexへの実装依頼: 2×2マスの大物装飾(footprint)対応

作成: Fable(デザイン担当) / 2026-07-10
対象: `Cafe Bloom - 花帖喫茶.html`(単一HTML、ビルド無し)
前提: v0.17.1 時点のコード。盤面は7×7(中央3×3が畑、草マスdeco-0〜39)

## ゴール

`GARDEN_DECOS` の項目に `footprint: 2` を付けると、その装飾が**草マス2×2(4マス)を占有**して、約2倍の大きさで描かれるようにする。

- 第一弾として `nijiizumi`(にじの泉)と `azumaya`(月映の東屋)に `footprint: 2` を設定し、`size` をそれぞれ **150** に変更する。
- SVGの中身・デザイン調整はFableが後でやる。今回は仕組みだけ。

## 設計方針(このとおりに)

### データ
- `decoPlaced` は今までどおり **アンカーのマスにだけ** `cellId → key` を持つ。占有マスは保存しない(実行時に計算)。書き出し/読み込みの互換はこれで保たれる。
- アンカーは2×2の**左上**(row最小・col最小)のマス。

### ヘルパー(新設)
- 草マスの列挙順(deco-N)と盤面座標(row,col)の相互変換関数。
  (renderIsoBoard内の列挙と同じ順序。既存の列挙ロジックを関数に切り出して共用すること。二重実装にしない)
- `decoFootprintCells(anchorCellId, footprint)` → 占有するcellIdの配列。
  範囲外・畑マス(中央3×3)を含む場合は null。
- `buildDecoOccupancy()` → Map(cellId → anchorCellId)。全placementの占有を展開。

### 置く(placeDeco)
- footprint 2 の場合: タップしたマスが2×2の「左上/右上/左下/右下」になる4通りのアンカー候補を順に試し、**4マス全部が空いている**(占有マップにもdecoPlacedにも無い)最初の候補に置く。
- どこにも置けない場合: `setNotice("ここには置けない。2×2マスの空きが要る。", "error")` で何もしない(在庫は減らさない)。

### 外す(removeDeco)
- アンカーを消せば占有も自然に消える(占有は計算値なので)。タップされたマスが占有マスだった場合もアンカーの装飾を外せること。

### 描画(箱庭ビュー)
- スプライトは**4マスの中心点**に、`size`(=150)で描く。接地は「4マスのうち一番手前(row+col最大)のマスのbaseY」を基準にする。
- 描画順: 一番手前のマスの位置で描く(奥の装飾や花に正しく重なるように)。
- 占有マスのヒット領域(`.iso-hit`)の `data-target` は**アンカーのcellId**にする(4マスのどこを押しても同じ装飾の詳細が開く)。
- 選択中の光る枠は4マス全体を囲う(2×2ぶんのひし形)。

### 描画(フラットビュー)
- グリッド構造は変えない。アンカーのセルに `grass-cell--big` クラスを付けて、CSSで `overflow: visible` ＋ `transform: scale(1.9)` ＋ `z-index` で大きく見せる(はみ出し表示でOK)。他の3マスは見た目は草のままで、クリックはアンカーと同じ扱い。

### 猫
- `pickCatCell` の候補から占有マスを除外(占有マップを使う)。

### 読み込み(normalizeGarden)
- footprint 2 のアンカーを検証: 4マスが盤面内・畑にかからない・他と重ならない(先勝ち)。
- **無効になった配置は消さずに `decoOwned` に戻す**(データを失わせない)。
- 既存セーブで1マス版の にじの泉 が置いてある場合も同じ扱い: そのマスをアンカーとして2×2が確保できればそのまま、無理ならポケットへ。

## 触ってはいけないもの
- `captureSettingsForm` 周りのガード(APIキー消失バグの再発防止)
- 各装飾の `svg` の中身、テーマ背景(isoBackdrop〜/isoAmbient〜)、蛍の層の仕組み
- `CafeBloom_無料版.html` の再生成、gitコミット・push(Fableが検証後にやる)
