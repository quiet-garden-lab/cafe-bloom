# Codexへの実装依頼: 畑の装飾エリア拡張(5×5 → 7×7)

作成: Fable(Claude Code側・デザイン担当) / 2026-07-09
対象: `Cafe Bloom - 花帖喫茶.html`(単一HTML、ビルド無し、これ1ファイルで完結)

## ゴール

畑の盤面を 5×5 から 7×7 に拡張する。

- 中央の 3×3(畑マス9個)はそのまま。位置は新しい盤面でも中央。
- 装飾を置ける草マスが 16個 → **40個** になる。
- 既存プレイヤーのセーブデータで、置いてある装飾が同じ見た目の位置に残ること(最重要)。

背景・装飾のデザイン変更はこの依頼に含めない(Fableが後で別途やる)。ロジックの拡張だけに絞ること。

## 現状の構造(読むべき場所)

- `renderFlatBoard()` / `renderIsoBoard()`: 5×5ループ。中央判定は `row >= 1 && row <= 3 && col >= 1 && col <= 3`、plotIndex は `(row - 1) * 3 + (col - 1)`。
- 草マスのIDは列挙順(row-major、中央スキップ)で `deco-0`〜`deco-15`。
- `normalizeGarden()` 内に検証正規表現 `^deco-([0-9]|1[0-5])$` が **2箇所**(decoPlaced と cat.cellId)。
- `randomEmptyDecoCell()` 相当の場所に `Array.from({ length: 16 }, (_, index) => \`deco-${index}\`)`。
- ISO描画定数: `ISO_TILE_W=96, ISO_TILE_H=48, ISO_VIEW_W=560, ISO_VIEW_H=396, ISO_ORIGIN_X=280, ISO_ORIGIN_Y=88`。
- フラット表示は `el.gardenBoard` にセルを25個appendしている。列数がCSS側(`grid-template-columns`等)にあるはずなので確認して7列に。

`deco-` で全文grepして、上記以外の参照箇所も漏れなく直すこと。

## 実装要件

1. **盤面**: ループを 0..6 に。畑マス判定は `row >= 2 && row <= 4 && col >= 2 && col <= 4`、plotIndex は `(row - 2) * 3 + (col - 2)`。
2. **草マスID**: 新しい列挙順で `deco-0`〜`deco-39`。検証正規表現を40個対応に更新(2箇所とも)。
3. **セーブ移行(最重要)**: 旧5×5の `deco-N` は、7×7では「内側リング」の同じ物理位置に対応する(旧(row,col) → 新(row+1,col+1))。
   - 旧インデックス→新インデックスの対応表を作り、読み込み時に `decoPlaced` のキーと `cat.cellId` を一度だけ変換する。
   - 移行済みフラグとして `garden.gridVersion = 2` を保存し、既に2なら変換しない。gridVersionが無いセーブ=旧版として扱う。
   - 変換は非破壊で(変換後の保存が成功するまで元データを壊さない、が理想。最低限、同じデータに二重変換がかからないこと)。
4. **ISO定数**: 7×7が収まるよう再計算(目安: VIEW_W=744, VIEW_H=492, ORIGIN_X=372, ORIGIN_Y=64 あたり。実際に表示して調整してよい)。
5. **フラット表示**: 7列グリッドに。セル49個。
6. **猫**: 移動先候補が40マスになるだけで、ロジックは触らない。

## 触ってはいけないもの
